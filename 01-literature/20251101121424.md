# [Kap 1.5.9]([[20250818102829]]) Hanois tårn
Hanois tårn er et godt eksempel på et problem som er mye lettere å løse rekursivt enn iterativt.
Vi begynner med å flytte de $n-1$ øverste brikkene til den midterste pinnen. Deretter flytter vi den nederste brikken til den borterste pinnen og gjentar trinn 1 for å ha flyttet hele tårnet. Det å flytte $n-1$ brikker er i seg selv det samme problemet, bare mindre. Vi kan derfor benytte nøyaktig samme metode rekursivt til vi sitter igjen med kun én brikke, noe vi vet hvordan vi flytter.
```java
/**
 * The following method recursively calculates the moves required to solve hanois tower in the fewest moves.
 * Each function call moves some number of discs to the help rod, then the bottom disc to the to rod, then the discs on the help rod back to the to rod.
 * The help rod is thus identical before and after each call.
 */
public static void HanoiTower(char from, char help, char to, int n) {
    if (n == 0) return; // Ingen brikker
    HanoiTower(from, to, help, n - 1);
    System.out.println("Brikke " + n + " fra " + from + " til " + to);
	HanoiTower(help, from, to, n - 1);
}
```
