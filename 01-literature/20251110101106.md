# [4.2.1]([[20250818102829]]) Queue
En kø er en datastruktur som følger FIFO-prinsippet. De elementene som har ligget i køen lengst behandles først.
```java
public interface Queue<T> {
    public boolean add(T value);
    public T peek();
    public T remove();
    public int size();
    public boolean isEmpty();
    public void clear();
}
```
I Java er metodene assosiert med en Queue et forbanna kaos. Dette skyldes muligheten for forskjellige feilhåndtering. Siden ingen blir enig, finnes det for eksempel flere metode for å hente ut, hvor noen returnerer null hvis køen er tom, noen kaster unntak, osv…
En liten oversikt:
| Formål                                   | Kaster unntak ved feil | Returnerer `null` ved feil | Beskrivelse                                                                                                            |
| :--------------------------------------- | :--------------------- | :------------------------- | :--------------------------------------------------------------------------------------------------------------------- |
| **Legge til (enqueue)**                  | `add(E e)`             | `offer(E e)`               | Legger til et element i køen. `add()` kaster `IllegalStateException` hvis full, `offer()` returnerer `false`.          |
| **Se på første element (uten å fjerne)** | `element()`            | `peek()`                   | Returnerer elementet foran i køen. `element()` kaster `NoSuchElementException` hvis tom, `peek()` returnerer `null`.   |
| **Fjerne første element (dequeue)**      | `remove()`             | `poll()`                   | Fjerner og returnerer første element. `remove()` kaster `NoSuchElementException` hvis tom, `poll()` returnerer `null`. |
